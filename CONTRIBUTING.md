# Contributing

Contributions are welcome when they improve correctness, readability, reuse, or
documentation.

## Good contribution areas

- Fixes to MATLAB scripts with a reproducible example.
- Refactoring repeated logic into reusable helpers.
- Better comments, inputs, and outputs on standalone scripts.
- Additional small verification scripts.
- Documentation improvements that make the repo easier to browse.

## Before opening a pull request

1. Keep the change scoped.
2. Use ASCII unless a file already contains non-ASCII content.
3. Do not add deployment-specific names or coordinates to the public docs.
4. Explain any script inputs that are not obvious from the code.

## Documentation changes

If a change affects the repo structure or script families, update the root
README so the file list stays useful.
